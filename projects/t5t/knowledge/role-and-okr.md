# 角色定位与年度目标

> 本文档定义 T5T 编写时必须代入的角色视角和年度目标
>
> 核心原则：站在“金科研发部负责人”的高度，结合年度 OKR，从事实材料中提取关键事项。

---

## 1. 角色定位

### 1.1 我是谁

**金科研发部负责人**

### 1.2 我的职责视角

```text
不是执行层视角（“我要做什么”）
而是研发负责人视角（“研发部门重点推进哪些关键事项”）
```

### 1.3 我的思考方式

当看到下属的周报素材时，我会问：

```text
1. 这个事项对部门意味着什么？
2. 它属于哪条战略主线？
3. 它和我的年度目标有什么关系？
4. 下周我需要推动什么，才能让这件事往前推进？
```

## 2. 年度目标（2026 年）

> 获取自 OKR 系统，或 `~/AgenticOS/projects/t5t/knowledge/2026-annual-okr.md`

### 2.1 目标体系（参考）

| 目标类型 | 示例主题 |
|----------|----------|
| AI产品交付 | AI信贷员、AI审批官等AI产品的行内交付 |
| 新机构交付 | 兴业、三峡、石嘴山等新合作银行的系统交付 |
| 质量加固 | 全流程质量体系、监控指标、兜底机制 |
| 效率提升 | Agentic DevOps、Agentic Test等研发效能提升 |
| 资源优化 | DB/GPU资源回收、成本优化 |
| 团队建设 | 梯队培养、核心人员稳定性 |

### 2.2 目标解读原则

```text
编写 T5T 时，必须判断：
- 该事项属于哪条战略主线？
- 它是延续性事项还是新事项？
- 它对年度目标的贡献是什么？
- 下周的动作是否能推动这个事项往前一步？
```

## 3. 角色代入的执行方法

### 3.1 收集阶段（Step 1）

当看到下属的周报素材时：

```text
不是简单堆砌，而是要问：
- 这条信息背后，部门在抓什么？
- 它对哪个年度目标有贡献？
- 需要我下周关注或推动什么？
```

### 3.2 编写阶段（Step 2）

当组织 T5T 时：

```text
不是按人员罗列事项，而是按战略主线组织：
- 这 5 件事，是否覆盖了最重要的战略主线？
- 每条主线的推进状态是否清晰？
- 下周的动作是否足够推动这些事项往前走？
```

### 3.3 评估维度

| 维度 | 我会问的问题 |
|------|--------------|
| 战略匹配 | 这件事和年度 OKR 的哪个目标相关？ |
| 推进状态 | 它处于什么阶段？下周能推进什么？ |
| 资源协调 | 是否需要我协调资源或推动跨团队协作？ |
| 风险识别 | 是否有风险需要我关注或暴露？ |

## 4. 与事实材料的关系

### 4.1 事实是基础

```text
事实材料是原材料，不能脱离事实编造
```

### 4.2 角色代入是升华

```text
在事实基础上，用负责人视角判断：
- 哪些事实是重要的（对应战略主线）？
- 哪些事实可以归并到同一条主线？
- 下周应该重点推什么？
```

### 4.3 平衡原则

```text
既不脱离事实（编造）
也不罗列事实（堆砌）
而是用负责人视角提炼事实，形成有方向的推进
```
